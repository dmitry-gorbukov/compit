### Что такое Asterisk?

**Asterisk** — это фреймворк с открытым исходным кодом для построения коммуникационных приложений. Проще говоря, это программное ядро, которое превращает обычный компьютер в полнофункциональную телефонную станцию (IP-АТС).

Он был создан в 1999 году Марком Спенсером и стал фундаментом для революции в телефонии, позволив создавать недорогие и гибкие решения на базе IP (VoIP).

**Ключевые возможности Asterisk:**

*   **IP-АТС:** Полнофункциональная замена дорогим аппаратным АТС.
*   **VoIP-шлюз:** Соединение традиционной телефонной сети (PSTN) с IP-сетями.
*   **IVR (Интерактивное голосовое меню):** Автосекретари ("Нажмите 1 для отдела продаж...").
*   **Конференц-связь:** Организация аудиоконференций.
*   **Голосовая почта:** С системой оповещения по email.
*   **Call-центр:** Функционал для организации колл-центров с очередями, записью разговоров и т.д.
*   **Поддержка множества протоколов:** SIP, IAX, H.323, MGCP и других.

### Как им пользоваться? С чего начать?

Использование Asterisk — это, по сути, **администрирование сервера**. Это не "пользовательское" приложение вроде Skype, а сложная система, требующая настройки. Вот путь, по которому стоит идти:

#### 1. Установка

Asterisk работает на Linux (рекомендуется Debian, CentOS, Ubuntu) и может быть собран из исходного кода. Это самый гибкий способ.

**Базовые шаги:**
```bash
# Пример для Ubuntu/Debian
# Обновление системы и установка зависимостей
sudo apt update
sudo apt install build-essential wget libssl-dev libncurses5-dev libnewt-dev libxml2-dev

# Скачивание исходного кода Asterisk (всегда берите стабильную версию с официального сайта)
wget http://downloads.asterisk.org/pub/telephony/asterisk/asterisk-20-current.tar.gz

# Распаковка, компиляция и установка
tar -xvf asterisk-20-current.tar.gz
cd asterisk-20*/
./configure
make
sudo make install
sudo make samples  # Установка стандартных конфигурационных файлов
```

**Для новичков** существуют готовые дистрибутивы, где Asterisk уже предустановлен и снабжен веб-интерфейсом для упрощения настройки:
*   **FreePBX** (самый популярный)
*   **Elastix**
*   **Issabel**

Они значительно упрощают начальные шаги.

#### 2. Базовая настройка (самое главное)

Конфигурация Asterisk происходит через текстовые файлы в директории `/etc/asterisk/`. Основные из них:

*   **`sip.conf`** — настройка SIP-аккаунтов (ваши внутренние телефоны и подключение к оператору связи).
*   **`extensions.conf`** — "мозг" системы. Здесь вы описываете логику: что происходит, когда кто-то набирает номер (диалект планов набора).
*   **`pjsip.conf`** — современная альтернатива `sip.conf` для более нового PJSIP-стека.
*   **`voicemail.conf`** — настройка голосовой почты.

**Простой пример: создание двух SIP-телефонов и их соединение.**

1.  **В файле `pjsip.conf`** определяем два телефона (6001 и 6002):
    ```ini
    [transport-udp]
    type=transport
    protocol=udp
    bind=0.0.0.0

    [6001]
    type=endpoint
    context=office-internal
    disallow=all
    allow=ulaw
    auth=auth6001

    [auth6001]
    type=auth
    auth_type=userpass
    password=mysecret1
    username=6001

    [6002]
    type=endpoint
    context=office-internal
    disallow=all
    allow=ulaw
    auth=auth6002

    [auth6002]
    type=auth
    auth_type=userpass
    password=mysecret2
    username=6002
    ```

2.  **В файле `extensions.conf`** описываем, что делать с вызовом:
    ```ini
    [office-internal]
    ; Когда набирают 6001 - позвонить на устройство 6001
    exten => 6001,1,Dial(PJSIP/6001)
    ; Когда набирают 6002 - позвонить на устройство 6002
    exten => 6002,1,Dial(PJSIP/6002)
    ```

#### 3. Запуск и управление

*   **Запуск:** `sudo systemctl start asterisk`
*   **Подключение к CLI (Командная строка):** `sudo asterisk -rvvv`
*   **Перезагрузка конфигурации:** В CLI выполнить `core reload` или `dialplan reload`.

CLI — это мощный инструмент для отладки, мониторинга звонков и управления системой в реальном времени.

#### 4. Тестирование

Установите на компьютер или смартфон SIP-клиент (например, **Zoiper**, **Linphone**, **MicroSIP**) и настройте в нем два аккаунта, используя данные из `pjsip.conf` (логин 6001, пароль mysecret1, адрес сервера — IP-адрес вашего сервера с Asterisk). После этого вы сможете звонить с 6001 на 6002 и наоборот.

### Для кого этот проект?

*   **IT-специалисты и системные администраторы**, которые хотят развернуть корпоративную АТС.
*   **Разработчики**, которые хотят создавать собственные голосовые приложения (IVR, колл-трекинг, интеграция с CRM).
*   **Энтузиасты и студенты**, изучающие телефонию и VoIP.
