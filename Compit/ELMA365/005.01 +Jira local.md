Отличное уточнение! Для локальной (on-premise) установки Jira процесс немного отличается от облачной версии.

## Генерация API-токена в локальной Jira

В локальной Jira **не используется** система API-токенов от Atlassian Account. Вместо этого используются:

### Способ 1: Личные токены доступа (для Jira 9.x и новее)

1. **Зайдите в Jira** под пользователем, от имени которого будет работать интеграция
2. **Нажмите на аватар** в правом верхнем углу
3. **Выберите "Управление аккаунтом" (Manage account)**
4. **В левом меню** перейдите в раздел **"Безопасность" (Security)**
5. **Найдите раздел "Личные токены доступа" (Personal access tokens)**
6. **Нажмите "Создать токен" (Create token)**
7. **Заполните данные:**
   - Имя токена (например, "ELMA365 Integration")
   - Срок действия (можно не ограничивать)
8. **Скопируйте токен** и сохраните в надежном месте

### Способ 2: Использование пароля приложения (для старых версий Jira)

Если в вашей версии Jira нет личных токенов:

1. **Зайдите в настройки профиля** (как в способе 1)
2. **Перейдите в "Безопасность"**
3. **Найдите "Пароли приложений" (App passwords)**
4. **Создайте новый пароль приложения**
5. **Используйте его вместо обычного пароля**

### Способ 3: Прямая аутентификация (Basic Auth)

Если токены недоступны, можно использовать прямую аутентификацию логин/пароль:

- **Логин:** username пользователя (не email!)
- **Пароль:** пароль учетной записи

**⚠️ Внимание!** Этот метод менее безопасен и может требовать дополнительных настроек.

## Проверка доступности API

Перед настройкой интеграции убедитесь, что REST API доступен:

1. **Откройте в браузере:** `http://your-jira-server/rest/api/2/project`
2. **Должны увидеть** список проектов в JSON-формате
3. **Если запрашивает авторизацию** - введите данные пользователя

## Настройка в ELMA365 для локальной Jira

В конфигурации соединения ELMA365 укажите:

```
Тип аутентификации: Basic Auth
Сервер: http://your-jira-server (или https://)
Логин: username_пользователя
Пароль: [Личный токен доступа или пароль приложения]
```

## Дополнительные настройки в Jira

Убедитесь, что пользователь для интеграции имеет права:
- **Глобальные права:** "Jira Users"
- **Права проекта:** "Browse Projects", "Create Issues", "Edit Issues" в нужных проектах

## Если токены не работают

В очень старых версиях Jira может потребоваться:

1. **Включить Basic Auth** в настройках безопасности
2. **Разрешить API доступ** для пользователя
3. **Проверить настройки CORS** если возникают ошибки при запросах

**Рекомендация:** По возможности обновите Jira до актуальной версии для использования более безопасных методов аутентификации через личные токены доступа.
